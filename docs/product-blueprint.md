# Product Blueprint

## 产品目标

构建一个支持上传 PDF、EPUB 等电子书格式的平台，重点服务技术类、英文类图书学习。平台按章节生成高保真学习解读，而不是只有薄摘要。

## 核心价值

- 降低技术书阅读启动成本
- 提高英文技术书吸收效率
- 保留原书论证链、术语上下文和代码讲解
- 把阅读变成带复习路径的学习流程

## 用户场景

### 1. 技术学习者

上传一本《Designing Data-Intensive Applications》这类英文技术书，希望先看每章导学，再决定精读顺序。

### 2. 工程团队内部学习

上传内部购买或有授权的技术材料，统一生成团队学习解读和讨论提纲。

### 3. AI 辅助陪学

让 Agent 根据章节自动输出术语解释、代码思路、提问清单、练习建议和复习卡片。

## 推荐能力拆分

### A. 文档摄取层

- 支持上传 `pdf`、`epub`、`md`
- OCR 兼容扫描版 PDF
- 目录识别
- 章节切分
- 页面和段落定位
- 图像、表格、代码块提取

### B. 学习生成层

- 每章生成低压缩导学
- 术语表
- 关键论点拆解
- 代码/公式/图表说明
- 难点解释
- 复习题与练习

### C. Agent 编排层

- 支持接入 Codex CLI
- 支持 Claude Code CLI
- 支持 DeepSeek API
- 支持模型路由和失败回退
- 支持结果缓存和人工复核

### D. 交互层

- 图书库
- 章节树
- 章节导学阅读页
- 术语联查
- 学习进度和复习记录

## “不过度压缩”的设计原则

平台不应只输出一页摘要，而应该输出分层结果：

1. `Quick Scan`
   用 1-3 分钟帮助用户判断本章在讲什么。
2. `Deep Explanation`
   逐节解释关键论点、关键术语、例子和代码。
3. `Retention Pack`
   输出复习题、误区提醒和后续实践建议。

## 技术建议

### Web

- Next.js 作为前端与 BFF 起点
- 后续可加对象存储、队列、数据库

### 解析

- EPUB 直接结构化读取
- PDF 走文本提取 + OCR + 目录识别
- 保留页面坐标，便于引用和回溯

### 生成

- 不直接把整章一次喂给模型
- 先做章节切块，再做多轮生成
- 对章节摘要设置最小覆盖要求
- 引入“覆盖率检查器”避免遗漏关键小节

## 第一阶段 MVP

### 用户可见

- 上传一本书
- 自动识别章节
- 点进任意章节，查看导学解释
- 可切换“简版 / 深版”

### 系统可见

- Book -> Chapter -> Section 数据模型
- 章节任务队列
- Agent 执行日志
- Prompt 模板版本化

## 后续建议

- 增加术语图谱
- 增加跨章节知识链接
- 增加双语对照学习模式
- 增加用户批注与个人笔记
